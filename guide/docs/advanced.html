第八章：高手进阶 —— 探索无限可能

1. 模型选择：逻辑还是性价比？

OpenCode 的一大魅力在于你可以随时更换“大脑”。中文社区的成熟经验是“混用策略”：

  • Claude 3.5 Sonnet: 逻辑之王 适合复杂重构、架构设计和疑难 Bug。
  • DeepSeek Coder V3: 性价比之选 适合日常增删改查、编写单测，通过本地或中转 API 接入，成本极低。

2. Oh My OpenCode 与 ULW 模式

如果你安装了 Oh My OpenCode 增强插件，你将解锁更强大的功能：

  • ULW (UltraWork) 模式: 开启多代理协作模式。当你需要完成一个跨模块的大型任务时，ULW 会自动调度研究、设计、编码等多个子智能体并行工作。
  • /ralph-loop: 进入“死磕模式”。智能体会不断尝试运行测试、修复报错，直到代码完全跑通为止。

3. Windows 用户避坑指南

  • 优先使用 WSL2：这是最顺畅的体验方式。
  • 证书错误：如果登录时遇到 TLS 报错，可尝试设置 set NODE_TLS_REJECT_UNAUTHORIZED=0。
  • 剪贴板权限：在 Linux 下如果无法粘贴，请确保安装了 xclip (X11) 或 wl-clipboard (Wayland)。

4. 定制化：Rules 与 Skills

这是让 OpenCode 真正“听懂人话”的关键。

Rules: 让 AI 遵守团队规范

在项目根目录创建 .opencode/rules.md。这是最简单但最有效的定制方式。 

# 项目编码规范

1. **语言要求**:
   - 必须使用 TypeScript 严格模式。
   - 所有变量名使用 camelCase。
   
2. **测试要求**:
   - 每次修改组件后，必须运行相关测试。
   - 必须使用 React Testing Library。

3. **提交规范**:
   - Commit message 必须遵循 Conventional Commits (feat, fix, docs)。

Skills: 给 AI 安装新技能

Skills 本质上是一段封装好的 Prompt 或脚本。你可以用 YAML 定义技能。 

# .opencode/skills/browser-test.md
---
name: browser-test
description: 使用 Playwright 运行浏览器测试
tools: [bash]
---
请执行以下步骤进行浏览器测试：
1. 运行 `npx playwright test $ARGS`
2. 如果失败，读取 `playwright-report/index.html` 并分析原因。

5. MCP 实战配置

以最常用的 PostgreSQL 和 Browser 插件为例： 

{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "postgresql://user:pass@localhost:5432/db"]
    },
    "browser": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-puppeteer"]
    }
  }
}

配置完成后，你就可以直接说：“查询 Users 表中最近注册的 10 个用户” 或 “访问 React 官网查一下最新的 Hook 文档”。

5. Oh My OpenCode 特色：ULW 模式

作为资深用户，你不能错过 Oh My OpenCode 插件带来的 UltraWork (ULW) 模式。它不仅是单兵作战，更是“特种部队”：

  • 多代理并发：同时调用 Explore、Librarian 和 Oracle 代理，速度提升 300%。
  • 深度代码库探索：使用 AST-grep 深入语法树级别理解你的代码。
